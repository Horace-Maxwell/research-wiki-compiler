# Code of Conduct

## Our Commitment

We want this project to be a respectful, technically honest, and low-ego place to collaborate.

Participants are expected to:

- act professionally and respectfully
- assume good intent while staying truthful about technical concerns
- welcome questions from people at different experience levels
- avoid harassment, intimidation, or personal attacks

## Expected Behavior

Examples of positive behavior include:

- giving clear, constructive technical feedback
- acknowledging uncertainty instead of bluffing
- documenting tradeoffs and risks openly
- respecting the local-first and review-first goals of the project

Examples of unacceptable behavior include:

- harassment or discriminatory language
- doxxing or sharing private information without consent
- aggressive, belittling, or hostile review behavior
- repeated bad-faith derailment of technical discussions

## Scope

This Code of Conduct applies to repository discussions, issues, pull requests, release notes, and any project-linked communication channels that may be added later.

## Enforcement

Project maintainers may remove, edit, or reject contributions or discussion content that violates this Code of Conduct.
