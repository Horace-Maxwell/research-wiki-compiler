# OpenClaw Walkthrough / OpenClaw 演示脚本

Use this as the fixed, reusable demo script for the official showcase.  
把这份文件当成官方 showcase 的固定、可复用演示脚本。

## Route order / 路由顺序

1. `/topics`
2. `/topics/openclaw`
3. `/questions?topic=openclaw`
4. `/sessions?topic=openclaw`
5. `/syntheses?topic=openclaw`
6. `/examples/openclaw`
7. `/settings`

## Step 1: `/topics`

- Click `OpenClaw`.  
  点击 `OpenClaw`。
- Say: the product starts by choosing a topic, not by dumping every file into a chat window.  
  讲法：这个产品从“选 topic”开始，而不是把所有文件直接丢进聊天框。
- Notice: the front door is small, direct, and oriented around the next topic-sized unit of work.  
  关注点：前门很小、很直接，而且是按 topic 这个工作单位来组织的。

## Step 2: `/topics/openclaw`

- Click the main question card first.  
  先点主问题卡片。
- Say: topic home is a cockpit, not a report. It foregrounds the next question, the next session, and the nearest synthesis.  
  讲法：topic home 是驾驶舱，不是报告页。它把下一个问题、下一次 session、最近的 synthesis 放在前面。
- Notice: supporting lanes stay secondary.  
  关注点：supporting lanes 仍然是次要的。

## Step 3: `/questions?topic=openclaw`

- Click through to the session view.  
  点击进入 session 页面。
- Say: this is a compact context pack, not a giant context dump.  
  讲法：这里给的是紧凑的 context pack，而不是巨大的上下文倾倒。
- Notice: the next problem is explicit and bounded.  
  关注点：下一个问题是明确而有边界的。

## Step 4: `/sessions?topic=openclaw`

- Click through to syntheses.  
  点击进入 syntheses。
- Say: sessions are resumable units of work, not vague “keep researching” loops.  
  讲法：session 是可恢复的工作单元，而不是模糊的“继续研究”循环。
- Notice: the product is managing practical working memory.  
  关注点：这个产品在管理的是实用工作记忆。

## Step 5: `/syntheses?topic=openclaw`

- Highlight one synthesis card before moving on.  
  在离开前点出一张 synthesis 卡片。
- Say: finished work gets promoted into durable judgment.  
  讲法：完成的工作会被提升成 durable judgment。
- Notice: the workflow is source -> note -> question -> session -> synthesis -> wiki.  
  关注点：整条工作链路是 source -> note -> question -> session -> synthesis -> wiki。

## Step 6: `/examples/openclaw`

- Open one featured page from `Start Here`.  
  从 `Start Here` 里打开一篇 featured page。
- Say: this rendered experience is still backed by the same canonical Markdown wiki.  
  讲法：这个渲染出来的体验，背后仍然是同一个 canonical Markdown wiki。
- Notice: the rendered app, the committed wiki, and the Obsidian vault all tell the same story.  
  关注点：渲染 app、已提交 wiki、Obsidian vault 讲的是同一个故事。
- Strong ending line: this is the difference between a durable knowledge workspace and a generic chat-with-files shell.  
  收束句：这就是 durable knowledge workspace 和普通 chat-with-files 外壳之间的差别。

## Step 7: `/settings`

- Switch the interface to Chinese once, reload, then switch back to English.  
  把界面切到中文，刷新一次，再切回英文。
- Say: bilingual support is part of the real product, not just the README.  
  讲法：双语支持是产品本体的一部分，不只是 README。
- Notice: language choice persists without breaking the main workflow.  
  关注点：语言选择会保持，而且不会破坏主工作流。
