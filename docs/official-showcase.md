## Why OpenClaw / 为什么是 OpenClaw

- It already proves the full chain from source to durable wiki.  
  它已经完整证明了从 source 到 durable wiki 的整条链路。
- It shows the main daily path clearly: topic home -> questions -> sessions -> syntheses.  
  它把默认日常主路径表达得很清楚：topic home -> questions -> sessions -> syntheses。
- It also keeps the supporting lanes honest instead of hiding uncertainty.  
  它也把 supporting lanes 诚实地保留下来，而不是假装没有不确定性。
- It aligns the rendered app, the canonical Markdown wiki, and the Obsidian vault around one knowledge base.  
  它让渲染后的 app、canonical Markdown wiki 和 Obsidian vault 都围绕同一套知识库对齐。
- It works in a reproducible reference mode and an optional live mode.  
  它同时支持可复现的 reference mode 和可选的 live mode。

## What This Showcase Proves / 这个案例证明了什么

- The product is not “chat with files.”  
  这个产品不是“拿文件聊天”。
- Good work becomes a durable, reviewable knowledge layer.  
  高价值工作会沉淀成可长期维护、可审阅的知识层。
- The working surfaces stay small and actionable instead of dumping giant context windows.  
  工作表面保持小而可行动，而不是把巨量上下文一次性倾倒出来。
- The wiki remains the truth layer even when the app and vault are the main reading surfaces.  
  即使 app 和 vault 是主要阅读界面，wiki 依然是最终真相层。

   打开 `/sessions?topic=openclaw`，展示这是可继续的有边界工作，而不是模糊的“继续研究”。
5. Open `/syntheses?topic=openclaw` and show how working knowledge hardens into durable judgment.  
   打开 `/syntheses?topic=openclaw`，展示工作中的知识如何固化成长期判断。

   最后打开 `/examples/openclaw`，再展示 canonical Markdown wiki 和 Obsidian vault。

## How To Reproduce / 怎么重现

### Official reference path / 官方 reference 路径

```bash

npm run env:check
npm run example:openclaw:reset
npm run example:openclaw:build
npm run example:openclaw:validate
npm run verify:routes

```

Or use the one-command path:  
或者直接用一条命令：

```bash
npm run showcase:openclaw
```

Reference mode is the official reproducible path. It does not require a provider key.  
reference mode 是官方可复现路径，不需要 provider key。

## Showcase Maintenance Contract / Showcase 维护合约

Treat OpenClaw as healthy only when all of these stay true:  
只有当这些条件都继续成立时，才把 OpenClaw 视为健康：

- the main product path still works  
  主产品路径仍然正常
- `/examples/openclaw` still renders from the committed showcase snapshot  
  `/examples/openclaw` 仍然基于提交进仓库的 showcase snapshot 渲染
- `npm run example:openclaw:validate` passes  
  `npm run example:openclaw:validate` 通过
- `npm run verify:routes` passes with zero HTML downloads and zero `pagePath` or `.html` href regressions  
  `npm run verify:routes` 通过，而且没有 HTML 下载、没有 `pagePath` 或 `.html` href 回归
- the committed wiki, rendered app, and Obsidian vault still describe the same knowledge base  
  已提交的 wiki、渲染 app、Obsidian vault 仍然描述同一套知识库

### What Must Stay Stable / 什么必须保持稳定

- Official route path:
  `/topics -> /topics/openclaw -> /questions?topic=openclaw -> /sessions?topic=openclaw -> /syntheses?topic=openclaw -> /examples/openclaw`
- The OpenClaw status as the flagship showcase, unless the repo intentionally replaces it
- The canonical wiki pages under `examples/openclaw-wiki/workspace/wiki/`
- Route safety: no download-like navigation, no raw `pagePath`, no raw `.html` app entry links

### Acceptable Drift / 可接受的漂移

- refreshed evaluation timestamps or scores after an intentional showcase rebuild  
  有意重建 showcase 后 evaluation 的时间戳或分数变化
- live-mode output from a real provider-backed run  
  live 模式下真实 provider 驱动的输出
- snapshot or baseline updates that were regenerated intentionally and then revalidated  
  有意重建并重新验证后的 snapshot 或 baseline 更新

### Unsafe Maintenance Shortcuts / 不安全的维护捷径

Do not do these:  
不要这样做：

- hand-edit the Obsidian vault when the canonical wiki should be the source change  
  当 canonical wiki 才是源头时，直接手改 Obsidian vault
- patch rendered workspace files under `tmp/` and treat them as lasting truth  
  手改 `tmp/` 下的 rendered workspace 并把它当成长期真相
- update docs before you have rerun the real showcase commands  
  在没重跑真实 showcase 命令前先改文档
- reintroduce raw file-style links into the main app path  
  把文件式原始链接重新带回主 app 路径

```bash
# set OPENAI_API_KEY

```

Live mode is for real provider-backed runs. It is useful, but not byte-for-byte stable.  
live mode 用于真实 provider 驱动的运行。它很有价值，但不承诺逐字节稳定。

## Why This Matters / 为什么这个 showcase 值得看

OpenClaw is the clearest proof that this system is useful for real knowledge work.  
OpenClaw 是这套系统对真实知识工作有价值的最清晰证明。

It shows a workflow where:  
它展示的是这样一种工作方式：

- sources become visible artifacts  
  source 先变成可见 artifact
- artifacts become reviewable work  
  artifact 再变成可审阅工作
- work becomes durable synthesis  
  工作再变成 durable synthesis
- synthesis stays reusable in both wiki and vault form  
  synthesis 最终以 wiki 和 vault 两种形式持续可复用

That is the difference between this product and a generic chat-with-files setup.  
这就是它和普通 chat-with-files 方案的真正差异。
