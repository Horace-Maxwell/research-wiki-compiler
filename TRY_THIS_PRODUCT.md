# Try This Product / 试用这个产品

This is the shortest external-facing guide for understanding why the product is useful.  
这是最短的对外试用指南，用来快速理解这个产品为什么有价值。

## What this is / 这是什么

Research Wiki Compiler turns source material into compact working context, bounded research work, durable syntheses, and a Markdown wiki that stays reviewable.  
Research Wiki Compiler 会把原始资料变成紧凑的工作上下文、有边界的研究工作、可长期复用的 synthesis，以及保持可审阅的 Markdown wiki。

## Why try it / 为什么值得试

- it is not generic chat-with-files  
  它不是普通的 chat-with-files
- it keeps context bundles small and explicit  
  它让上下文包保持小而明确
- it turns good work into durable notes and syntheses  
  它把高价值工作沉淀成 durable note 和 synthesis
- it gives you a clear next action instead of one giant dump  
  它给你的是清晰的下一步，而不是一个巨大的信息堆

## Fastest local path / 最快本地路径

```bash
npm install
npm run env:check
npm run demo:reset
npm run showcase:openclaw
npm run dev
```

Then open:  
然后打开：

`/topics -> /topics/openclaw -> /questions?topic=openclaw -> /sessions?topic=openclaw -> /syntheses?topic=openclaw -> /examples/openclaw`

## What to notice / 应该注意什么

- `/topics`: the product starts by choosing a topic, not by loading every file  
  `/topics`：产品从选 topic 开始，而不是把所有文件一次性塞进去
- `/topics/openclaw`: topic home is a cockpit for the next action  
  `/topics/openclaw`：topic home 是服务“下一步动作”的驾驶舱
- `/questions` and `/sessions`: research work is bounded and resumable  
  `/questions` 和 `/sessions`：研究工作是有边界、可恢复的
- `/syntheses`: good work hardens into durable judgment  
  `/syntheses`：高价值工作会固化成 durable judgment
- `/examples/openclaw`: the rendered experience still points back to the same canonical wiki  
  `/examples/openclaw`：渲染出来的体验依然指向同一个 canonical wiki
