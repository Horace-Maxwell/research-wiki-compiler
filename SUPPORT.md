## GitHub Workflow

For the current GitHub-hosted workflow:

- reproducible bugs should go to GitHub Issues
- feature requests should go to GitHub Discussions when enabled, or to the feature request template if Discussions are not enabled yet
- security reports should follow [SECURITY.md](SECURITY.md), not public issue trackers

## Security Issues

If the issue may expose data, credentials, or unsafe mutation behavior, follow [SECURITY.md](SECURITY.md) instead of a public bug report.

## What Helps Most

When asking for help, call out which product loop you are in:

- compile
- review
- ask
- archive
- audit

Include:

- exact steps to reproduce
- workspace mode and provider mode if relevant
- expected behavior versus actual behavior
- logs, screenshots, or sample non-sensitive artifacts when possible
